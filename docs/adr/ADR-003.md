# ADR-003: Spring Events (Observer) para notificaciones desacopladas

## Estado
Aceptado — 2026-05-20

## Contexto
El servicio de pedidos necesita notificar a múltiples receptores
(email, log, futuro SMS) cuando un pedido es procesado. El
acoplamiento directo a cada canal hace frágil el servicio central
y mezcla responsabilidades de negocio con infraestructura.

## Decisión
Se usa Spring ApplicationEventPublisher (implementación de Observer)
para publicar PedidoProcesadoEvent. Cada canal de notificación
implementa un @EventListener independiente (NotificacionEmail,
NotificacionLog). FachadaPedidos solo publica el evento sin conocer
quién escucha.

## Consecuencias

**Positivas:**
- Agregar un nuevo canal de notificación no requiere modificar
  FachadaPedidos ni ninguna clase existente.
- Los listeners son opcionales: si no hay suscriptores, el evento
  se ignora sin error.

**Negativas:**
- El orden de ejecución de listeners no está garantizado.
- El debugging es más complejo: hay que rastrear listeners para
  entender qué ocurre tras publicar el evento.
